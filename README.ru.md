# <img src="extension/icons/icon128.png" width="32" alt="" align="top"> Chromium Bridge

[English version](README.md)

Мост между Chromium-браузером и Claude Code. Официальное расширение «Claude in Chrome»
в некоторых Chromium-браузерах (Arc, Vivaldi и др.) подключается, но автоматизация виснет:
его инструменты строятся на tab groups API, которого там нет или который не работает.
Этот мост использует только обычные `chrome.tabs` / `chrome.scripting` / `chrome.debugger`,
поэтому работает в любом Chromium-браузере, куда можно поставить расширение.

## Демо

Claude управляет браузером через мост — открывает Wikipedia, вводит поисковый
запрос и переходит к статье:

![Демо Chromium Bridge: Claude управляет Chromium-браузером](docs/demo.gif)

## Архитектура

```
Claude Code ⇄ (stdio MCP) ⇄ server/index.mjs ⇄ (WebSocket, 127.0.0.1:8929) ⇄ расширение в браузере
                                   ⇅ (WebSocket /chat)
                            панель чата (popup у иконки расширения)
```

- `extension/` — MV3-расширение: service worker держит WebSocket к локальному серверу
  и исполняет команды (вкладки, навигация, текст страницы, скриншот, клик, заполнение полей).
  Клик по иконке открывает панель чата (`chat.html`) — попап у иконки расширения.
- `server/` — MCP-сервер (stdio), отдаёт Claude Code инструменты `browser_*`
  и проксирует их в расширение. Принимает WS-соединения только с Origin
  `chrome-extension://…` — обычные веб-страницы подключиться не могут.
  Дополнительно обслуживает канал `/chat`: сообщения панели прогоняются через
  Claude Agent SDK (авторизация — логин Claude Code) с теми же `browser_*`-инструментами,
  встроенные инструменты (Bash, Read и т.д.) отключены.

## Панель чата

Аналог боковой панели «Claude in Chrome»: попап, открывающийся по клику
на иконку расширения (без `chrome.sidePanel` — он поддерживается не везде).
Чат видит браузер: может смотреть вкладки, читать страницы, делать скриншоты и кликать.

![Демо панели чата: Claude открывает статью Википедии и отвечает по ней](docs/chat-demo.gif)

Интерфейс панели — английский по умолчанию; если язык интерфейса браузера русский,
панель автоматически переключается на русский. Селектор языка (Auto / English /
Русский) в нижней строке панели переопределяет автоопределение; бейдж на странице
следует тому же выбору.

- Попап закрывается при потере фокуса (клик по странице) — это поведение браузера.
  Контекст разговора при этом не теряется: панель помнит session_id,
  сервер продолжает разговор через `resume` Agent SDK. Но идущий ход агента
  при закрытии попапа прерывается.
- Работает, только когда запущен сервер (обычно — активная сессия Claude Code
  с MCP `chromium-bridge`); иначе панель покажет «Сервер недоступен».
- Выбор модели — в шапке панели: «По умолчанию» берёт модель из
  `~/.claude/settings.json` (то, что задано через `/model`; SDK-сессии сами
  настройки Claude Code не читают, сервер передаёт модель явно), остальные
  пункты — жёсткий выбор. Смена применяется на лету (`setModel`) и запоминается.
  Стартовое переопределение: `CHROMIUM_BRIDGE_CHAT_MODEL=sonnet` в окружении сервера.
  Порт: `CHROMIUM_BRIDGE_PORT` (по умолчанию 8929).
- История чатов: кнопка 🕓 в шапке — список прошлых разговоров (хранятся
  в localStorage панели, последние 30).
- После каждого хода — строка расходов: токены хода (↑ ввод с кэшем / ↓ вывод)
  и накопленная стоимость сессии в $ (при подписке это расчётная величина,
  не отдельный счёт).
- Скриншоты, которые агент делает по ходу работы, показываются прямо в ленте
  чата (клик — развернуть). В историю не сохраняются (localStorage не резиновый).
- В чат можно вставлять картинки из буфера (Cmd+V в поле ввода, до 5 за
  сообщение) — модель их видит; в истории остаётся только пометка.
- Режим «Спрашивать перед действием» (галка над полем ввода): чтение
  (вкладки, текст, скриншоты, консоль, сеть) идёт без вопросов, а мутирующие
  действия — клики/ввод/навигация/JS/формы/закрытие вкладок/загрузка файлов —
  ждут подтверждения в карточке «Разрешить / Отклонить». Отклонение агент
  видит и продолжает разговор. Переключение действует сразу, без пересоздания
  сессии (через canUseTool Agent SDK).

## Индикация на странице

Когда Claude действует на вкладке (из панели или из Claude Code):

- по краям страницы горит оранжевое свечение с бейджем «✳ Claude работает…»,
  гаснет через 2,5 с после последнего действия;
- виртуальный курсор (оранжевая стрелка) плавно едет к точке действия,
  при клике расходится пульс-кольцо; исчезает через 3,5 с бездействия.

На скриншотах и то и другое прячется, чтобы не попадать в кадр и не сбивать
модель при работе с координатами. На страницах, куда нельзя внедрять скрипты
(`chrome://` и т.п.), индикация молча не показывается.

## Установка

1. **Расширение**: склонировать репозиторий, открыть `chrome://extensions`
   (в нужном пространстве/профиле!), включить «Developer mode», нажать
   «Load unpacked» и выбрать папку `extension/`.
2. **MCP-сервер** — любой из способов:
   - через npm: `claude mcp add -s user chromium-bridge -- npx chromium-bridge`
   - из клона: `cd server && npm install`, затем
     `claude mcp add -s user chromium-bridge -- node "$(pwd)/index.mjs"`.

   Загружается при старте сессии — после установки расширения перезапустить сессию Claude Code.
3. Проверка: инструмент `browser_status` должен вернуть `{"connected": true}`.

## Инструменты (v0.5)

| Инструмент | Что делает |
|---|---|
| `browser_status` | Проверка соединения с расширением |
| `browser_tabs_list` | Список вкладок (id, заголовок, URL) |
| `browser_tab_create` / `browser_tab_close` | Открыть / закрыть вкладку |
| `browser_navigate` | Перейти по URL; `back`/`forward` — история |
| `browser_page_text` | Заголовок, URL и видимый текст страницы |
| `browser_computer` | Мышь/клавиатура/скриншоты через CDP: клики по координатам или ref, drag, hover, type, key-сочетания, scroll, zoom-скриншот региона, wait |
| `browser_read_page` | Accessibility-дерево с ref-идентификаторами (filter=interactive) |
| `browser_find` | Поиск элементов по тексту/роли, возвращает ref |
| `browser_form_input` | Значение input/textarea/select/checkbox/contenteditable по selector или ref |
| `browser_click` | DOM-клик по CSS-селектору (простой .click()) |
| `browser_upload_file` | Подставить файлы в `<input type="file">` |
| `browser_javascript` | Выполнить JS на странице (await поддерживается) |
| `browser_console_messages` | Консоль вкладки (с regex-фильтром) |
| `browser_network_requests` | Сетевые запросы вкладки (с regex-фильтром) |
| `browser_resize_window` | Размер окна |
| `browser_gif_start` / `browser_gif_stop` | Запись GIF вкладки → файл; на длинных записях частота кадров автоматически снижается, чтобы влез весь сценарий |

Всё, кроме базовых вкладочных операций, работает через `chrome.debugger` (CDP):
скриншоты не требуют активации вкладки, клики — настоящие события мыши,
консоль и сеть собираются с момента первого CDP-обращения к вкладке.

## Ограничения

- Расширение живёт в одном профиле браузера — ставить в тот, где нужна автоматизация.
- Service worker может засыпать; keepalive-alarm будит его каждые ~30 секунд,
  сервер ждёт переподключения до 12 секунд перед ошибкой.
- При первом CDP-действии браузер показывает плашку «Chromium Bridge начал отладку
  этого браузера» — это нормально, debugger и есть механизм управления.
  Закрытие плашки отключает debugger (следующее действие подключит заново).
- Консоль/сеть не пишутся ретроактивно — только после первого обращения к вкладке.
- Порт 8929 занят одной сессией: вторая параллельная сессия Claude Code
  не сможет поднять свой WS-сервер (расширение останется у первой).
